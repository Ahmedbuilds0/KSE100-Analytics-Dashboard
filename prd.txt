# PSX KSE-100 Comparative Analytics Dashboard & Speed Analyzer

## Overview

Build a modern JavaFX desktop application that loads historical KSE-100 Index data from a local text file, validates and repairs corrupted records, visualizes the data, applies two different stock market analysis algorithms, and compares their execution speeds using Java's System.nanoTime().

The application should look professional, modern, and suitable for a financial analytics dashboard.

# Dataset

The application must read data from:

```text
kse100_project_dataset.txt
```

Dataset contains approximately 3 years of KSE-100 historical data.

Format:

```text
2023-01-02,40350.25
2023-01-03,40510.80
2023-01-04,40475.60
```

Some records intentionally contain corrupted values:

```text
2023-05-12,abc
2024-01-15,
2024-07-10,N/A
2025-03-05,invalid_number
```

---

# Data Validation

When reading the file:

* Detect invalid numeric values.
* Detect empty values.
* Detect corrupted text values.

Use try-catch blocks.

When corruption is detected:

* Replace the value with the previous valid value.
* Continue processing.
* Never crash the application.

# Algorithm 1

## Simple Moving Average (SMA)

Purpose:

Smooth market fluctuations and noise.

Formula:

(Current + Previous + Next) / 3

Output:

Smoothed KSE-100 trend line.

---

# Algorithm 2

## Exponential Moving Average (EMA)

Purpose:

Provide a more responsive trend indicator.

Formula:

EMA = Current × α + PreviousEMA × (1 − α)

Use:

α = 0.2

Output:

EMA trend line.

---

# Performance Measurement

Measure execution speed of both algorithms using:

```java
System.nanoTime()
```

Implementation:

```java
long startTime = System.nanoTime();

/* processing */

long endTime = System.nanoTime();

long executionTime = endTime - startTime;
```

Display:

```text
SMA Speed: 42000 ns
EMA Speed: 38500 ns
```

Both values must remain visible simultaneously.
